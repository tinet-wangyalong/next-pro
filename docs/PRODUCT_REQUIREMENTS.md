# 产品需求文档：TeamPilot

## 1. 产品定位

TeamPilot 是一个面向小团队的 SaaS 工作台，用来管理团队知识、任务记录和 AI 辅助问答。AI 在本项目中不是替代开发者写代码的核心，而是产品功能的一部分：用户可以把团队资料沉淀到组织空间中，并通过 AI 助手查询、总结和生成行动建议。

这个产品适合作为学习型 SaaS 项目，因为它能自然覆盖用户系统、多租户、权限、数据库建模、Redis 队列、异步任务、Docker、本地开发、API 设计和前端状态管理。

## 2. 目标用户

- [ ] 独立开发者：想管理个人项目资料、任务和 AI 对话记录。
- [ ] 小型团队：希望在一个工作台里整理团队知识和项目任务。
- [ ] 学习者：希望通过真实 SaaS 场景练习全栈工程能力。

## 3. 核心价值

- [ ] 把组织、成员、知识条目、任务和 AI 对话统一放在一个 SaaS 骨架里。
- [ ] 通过组织空间实现多租户隔离。
- [ ] 通过角色权限练习真实业务权限设计。
- [ ] 通过 AI 任务队列练习异步任务和状态追踪。
- [ ] 通过清晰的页面和 API 边界，支持逐步手写实现。

## 4. 版本规划

### V0：项目骨架

- [ ] 初始化 Monorepo。
- [ ] 启动 Next.js 前端。
- [ ] 启动 NestJS 后端。
- [ ] 使用 Docker Compose 启动 PostgreSQL 和 Redis。
- [ ] API 提供健康检查接口。
- [ ] 前端提供基础首页。

### V1：账号与组织

- [ ] 用户注册。
- [ ] 用户登录。
- [ ] JWT 鉴权。
- [ ] 创建组织。
- [ ] 查看我的组织。
- [ ] 切换当前组织。
- [ ] 查看组织成员。
- [ ] 基础角色：Owner、Admin、Member。

### V2：团队知识库

- [ ] 创建知识条目。
- [ ] 编辑知识条目。
- [ ] 删除知识条目。
- [ ] 按标题和内容搜索知识条目。
- [ ] 按组织隔离知识条目。
- [ ] 记录创建人和最后更新人。

### V3：任务管理

- [ ] 创建任务。
- [ ] 更新任务状态。
- [ ] 指派任务负责人。
- [ ] 设置任务优先级。
- [ ] 按状态、负责人、优先级筛选任务。
- [ ] 在仪表盘展示任务概览。

### V4：AI 助手

- [ ] 创建 AI 会话。
- [ ] 发送用户消息。
- [ ] 保存 AI 回复。
- [ ] 使用 Redis + BullMQ 处理 AI 回复任务。
- [ ] 支持 mock AI provider，先不依赖真实模型。
- [ ] 后续预留 OpenAI API provider。
- [ ] AI 可以基于当前组织的知识条目做简单回答。

### V5：SaaS 增强

- [ ] 成员邀请。
- [ ] 审计日志。
- [ ] 套餐和订阅预留。
- [ ] 用量统计。
- [ ] 限流。
- [ ] 测试覆盖。
- [ ] 生产部署文档。

## 8. 关键用户流程

### 注册并创建组织

- [ ] 用户进入注册页。
- [ ] 输入邮箱、用户名、密码。
- [ ] 注册成功后自动创建默认组织。
- [ ] 用户进入仪表盘。

### 登录并进入组织

- [ ] 用户进入登录页。
- [ ] 输入邮箱和密码。
- [ ] 登录成功后获取 token。
- [ ] 前端保存登录状态。
- [ ] 用户进入上次访问的组织，或进入默认组织。

### 创建知识条目

- [ ] 用户进入知识库。
- [ ] 点击新建。
- [ ] 输入标题、正文和标签。
- [ ] 保存后进入详情页。
- [ ] 系统写入审计日志。

### 创建任务

- [ ] 用户进入任务列表。
- [ ] 点击新建任务。
- [ ] 输入标题、描述、负责人、优先级和截止时间。
- [ ] 保存后任务出现在列表。
- [ ] 仪表盘统计同步更新。

### 使用 AI 助手

- [ ] 用户进入 AI 助手页。
- [ ] 创建或选择一个会话。
- [ ] 输入问题。
- [ ] 后端保存用户消息。
- [ ] 后端创建异步 AI 任务。
- [ ] 前端展示消息处理中状态。
- [ ] worker 生成 mock AI 回复。
- [ ] 前端轮询或刷新后展示回复。

## 9. 非功能需求

- [ ] 所有组织数据必须按 `organizationId` 隔离。
- [ ] 所有受保护接口必须验证登录状态。
- [ ] 修改类接口必须检查角色权限。
- [ ] API 错误响应格式保持一致。
- [ ] 所有密码必须哈希保存。
- [ ] `.env` 不提交真实密钥。
- [ ] Redis 任务失败时必须记录错误原因。
- [ ] 前端页面必须有 loading、empty、error 状态。
- [ ] 关键业务操作写入审计日志。

## 10. 暂不实现

- [ ] 在线支付。
- [ ] 真实邮件发送。
- [ ] 复杂富文本编辑器。
- [ ] 文件上传。
- [ ] 企业级 SSO。
- [ ] 复杂向量检索。
- [ ] 真实多模型路由。

这些能力可以作为后续扩展，但 V1 到 V4 阶段先不要引入，避免学习路径过重。
