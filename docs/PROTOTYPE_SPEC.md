# 页面原型与交互说明：TeamPilot

本文档是低保真原型说明，用来指导手写前端页面。实现时优先保证页面结构、交互流程和状态完整，不追求复杂视觉设计。

## 1. 全局布局

### 登录前布局

- [ ] 页面居中显示表单区域。
- [ ] 顶部显示产品名 `TeamPilot`。
- [ ] 表单下方提供登录和注册之间的跳转。
- [ ] 错误信息显示在表单按钮上方。

### 登录后布局

- [ ] 左侧固定导航栏。
- [ ] 顶部显示当前组织、用户菜单和退出登录。
- [ ] 主区域显示当前页面内容。
- [ ] 移动端可以先简化为顶部导航。

### 左侧导航

- [ ] Dashboard：仪表盘。
- [ ] Knowledge：知识库。
- [ ] Tasks：任务。
- [ ] AI Assistant：AI 助手。
- [ ] Members：成员。
- [ ] Settings：设置。
- [ ] Audit Logs：审计日志。

## 2. 登录页

### 页面内容

- [ ] 产品名：TeamPilot。
- [ ] 标题：登录到你的工作台。
- [ ] 邮箱输入框。
- [ ] 密码输入框。
- [ ] 登录按钮。
- [ ] 注册入口。

### 状态

- [ ] 默认状态：按钮可点击。
- [ ] 提交中：按钮显示加载中并禁用。
- [ ] 错误状态：显示账号或密码错误。
- [ ] 成功状态：跳转到仪表盘。

### 验收标准

- [ ] 空邮箱不能提交。
- [ ] 空密码不能提交。
- [ ] 登录失败有明确错误提示。
- [ ] 登录成功后保存 token。

## 3. 注册页

### 页面内容

- [ ] 产品名：TeamPilot。
- [ ] 标题：创建你的团队工作台。
- [ ] 用户名输入框。
- [ ] 邮箱输入框。
- [ ] 密码输入框。
- [ ] 确认密码输入框。
- [ ] 注册按钮。
- [ ] 登录入口。

### 状态

- [ ] 密码不一致时提示错误。
- [ ] 注册中禁用按钮。
- [ ] 注册成功后进入仪表盘。

### 验收标准

- [ ] 注册成功后自动创建默认组织。
- [ ] 注册成功后自动登录或跳转登录页，两种方式任选一种并保持一致。

## 4. 仪表盘

### 页面内容

- [ ] 顶部显示欢迎语。
- [ ] 显示当前组织名称。
- [ ] 四个统计卡片：
  - [ ] 知识条目数。
  - [ ] 待办任务数。
  - [ ] 进行中任务数。
  - [ ] AI 会话数。
- [ ] 最近任务列表。
- [ ] 最近知识条目列表。

### 空状态

- [ ] 没有任务时显示创建任务入口。
- [ ] 没有知识条目时显示创建知识入口。

### 验收标准

- [ ] 登录用户可以看到自己当前组织的数据。
- [ ] 统计数据不会包含其他组织的数据。

## 5. 组织切换

### 页面内容

- [ ] 当前组织名称。
- [ ] 组织下拉选择器。
- [ ] 创建新组织按钮。

### 交互

- [ ] 切换组织后刷新当前页面数据。
- [ ] 创建新组织后自动切换到新组织。

### 验收标准

- [ ] 只能看到自己加入的组织。
- [ ] 切换组织后知识库、任务、成员列表都随之变化。

## 6. 成员管理页

### 页面内容

- [ ] 成员列表表格。
- [ ] 列：用户名、邮箱、角色、加入时间、操作。
- [ ] 邀请成员按钮。
- [ ] 修改角色操作。
- [ ] 移除成员操作。

### 权限

- [ ] Owner 可以修改所有成员角色。
- [ ] Owner 可以移除 Admin 和 Member。
- [ ] Admin 可以邀请成员。
- [ ] Member 只能查看成员列表。

### 验收标准

- [ ] 无权限操作按钮不显示或禁用。
- [ ] 权限不足时后端也必须拒绝请求。

## 7. 知识库列表页

### 页面内容

- [ ] 页面标题：知识库。
- [ ] 新建知识按钮。
- [ ] 搜索框。
- [ ] 标签筛选。
- [ ] 知识条目列表。
- [ ] 每条显示标题、摘要、标签、更新时间、创建人。

### 状态

- [ ] 加载状态：显示骨架或加载文案。
- [ ] 空状态：提示创建第一条知识。
- [ ] 错误状态：提供重试按钮。

### 验收标准

- [ ] 搜索只返回当前组织的数据。
- [ ] 点击条目进入详情页。

## 8. 知识条目详情页

### 页面内容

- [ ] 标题。
- [ ] 正文。
- [ ] 标签。
- [ ] 创建人。
- [ ] 最后更新时间。
- [ ] 编辑按钮。
- [ ] 删除按钮。

### 交互

- [ ] 编辑时进入表单模式。
- [ ] 保存成功后回到详情模式。
- [ ] 删除前需要确认。

### 验收标准

- [ ] 无权限用户不能编辑或删除。
- [ ] 删除后回到知识库列表。

## 9. 任务列表页

### 页面内容

- [ ] 页面标题：任务。
- [ ] 新建任务按钮。
- [ ] 状态筛选。
- [ ] 负责人筛选。
- [ ] 优先级筛选。
- [ ] 任务列表或看板。

### 任务卡片

- [ ] 标题。
- [ ] 状态。
- [ ] 优先级。
- [ ] 负责人。
- [ ] 截止时间。

### 验收标准

- [ ] 可以创建任务。
- [ ] 可以更新任务状态。
- [ ] 筛选条件生效。

## 10. 任务详情页

### 页面内容

- [ ] 任务标题。
- [ ] 任务描述。
- [ ] 状态选择器。
- [ ] 优先级选择器。
- [ ] 负责人选择器。
- [ ] 截止时间。
- [ ] 创建人。
- [ ] 更新时间。

### 交互

- [ ] 修改状态后保存。
- [ ] 修改负责人后保存。
- [ ] 修改失败时回滚或显示错误。

### 验收标准

- [ ] 负责人只能从当前组织成员中选择。
- [ ] 状态更新后列表页同步变化。

## 11. AI 助手页

### 页面结构

- [ ] 左侧会话列表。
- [ ] 右侧聊天区域。
- [ ] 底部消息输入框。
- [ ] 发送按钮。

### 会话列表

- [ ] 新建会话按钮。
- [ ] 会话标题。
- [ ] 最近更新时间。
- [ ] 当前会话高亮。

### 聊天区域

- [ ] 用户消息靠右。
- [ ] AI 消息靠左。
- [ ] 处理中消息显示状态。
- [ ] 失败消息显示重试入口。

### 交互流程

- [ ] 用户输入问题。
- [ ] 点击发送。
- [ ] 立即展示用户消息。
- [ ] 展示 AI 处理中占位消息。
- [ ] 后端任务完成后展示 AI 回复。

### 验收标准

- [ ] 消息会保存到数据库。
- [ ] 刷新页面后历史消息仍然存在。
- [ ] AI 回复可以先使用 mock 内容。
- [ ] 任务失败时页面能显示失败状态。

## 12. 组织设置页

### 页面内容

- [ ] 组织名称输入框。
- [ ] 组织 slug 输入框。
- [ ] 保存按钮。
- [ ] 危险区域：删除组织。

### 权限

- [ ] Owner 可以修改组织设置。
- [ ] Admin 和 Member 只能查看或无权访问。

### 验收标准

- [ ] 修改组织名称后导航中的名称同步更新。
- [ ] 删除组织必须二次确认。

## 13. 审计日志页

### 页面内容

- [ ] 操作时间。
- [ ] 操作人。
- [ ] 操作类型。
- [ ] 操作对象。
- [ ] 变更摘要。

### 筛选

- [ ] 按操作人筛选。
- [ ] 按操作类型筛选。
- [ ] 按时间范围筛选。

### 验收标准

- [ ] 创建、修改、删除知识条目会记录日志。
- [ ] 成员角色变更会记录日志。
- [ ] 任务状态变更会记录日志。

## 14. 推荐实现顺序

- [ ] 先实现登录前布局。
- [ ] 再实现登录和注册页。
- [ ] 再实现登录后布局和仪表盘。
- [ ] 再实现组织切换。
- [ ] 再实现知识库。
- [ ] 再实现任务。
- [ ] 再实现 AI 助手。
- [ ] 最后补成员、设置和审计日志。

## 15. 手写实现建议

- [ ] 每个页面先写静态 UI。
- [ ] 静态 UI 完成后再接真实 API。
- [ ] 每次只接一个接口。
- [ ] 每个接口都处理 loading、success、error。
- [ ] 每做完一个页面，在文档中勾选完成项。
- [ ] 不熟悉的概念先写最小版本，再逐步重构。
